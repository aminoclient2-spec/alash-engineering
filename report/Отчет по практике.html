1. Введение

1.1. Цель и задачи производственной практики

Производственная практика направлена на закрепление теоретических знаний, полученных в рамках учебного плана по специальности «Computer Science», и их применение в условиях реального проектного процесса. Цель практики — приобрести практические навыки веб-разработки, научиться работать с современными инструментами фронтенда, освоить процесс создания многостраничного веб-сайта от проектирования до публикации.

Основные задачи:

  • Анализ требований и формирование технического задания;
  • Проектирование структуры сайта и пользовательского интерфейса;
  • Реализация адаптивной вёрстки на HTML, CSS и JavaScript;
  • Интеграция CSS-фреймворков и JS-библиотек;
  • Реализация мультиязычности и мультивалютности;
  • Публикация проекта на хостинге и демонстрация заказчику.

1.2. Информация о компании, где проходила практика

Практика проходила в компании ALASH ENGINEERING CORPORATION — казахстанском производителе компьютерной техники и периферийных устройств. Компания занимается сборкой ПК, разработкой периферийного оборудования (клавиатуры, мыши, наушники, веб-камеры) и продажей готовых решений для офиса, дома и профессионального использования. Основной стек технологий компании включает HTML, CSS, JavaScript, Bootstrap, Figma, а также инструменты командной работы: Git и GitHub.

1.3. Моя роль в команде и область ответственности

Я выполнял роль frontend-разработчика и веб-верстальщика. В мои задачи входили:

  • Изучение технического задания и пожеланий заказчика;
  • Проектирование архитектуры сайта и навигации;
  • Вёрстка всех страниц сайта на HTML, CSS и JavaScript;
  • Реализация адаптивности и поддержки трёх языков (KZ, RU, EN);
  • Разработка функциональных компонентов: корзина, избранное, конфигуратор ПК, поиск, чат;
  • Взаимодействие с командой и синхронизация через Git/GitHub.

1.4. Технологии, использованные в ходе практики

В рамках практики я использовал следующие инструменты и технологии:

  • Figma — для создания макетов и прототипов интерфейса;
  • HTML5 — для построения семантической структуры страниц;

  • JavaScript (ES6) — для интерактивности: корзина, поиск, фильтры, конфигуратор;
  • Bootstrap 5 — для ускорения разработки с использованием готовых компонентов;
  • Font Awesome 6 — для иконок социальных сетей и интерфейса;

2. Анализ технического задания (ТЗ)

2.1. Потребности заказчика: сайт для компании ALASH ENGINEERING

Заказчик — компания ALASH ENGINEERING CORPORATION, производящая компьютеры и периферию в Казахстане. Цель сайта — презентация продукции компании, предоставление возможности конфигурации ПК, привлечение клиентов из Казахстана и стран СНГ, обеспечение удобного интерфейса для выбора и заказа товаров.

2.2. Целевая аудитория и задачи сайта

Целевая аудитория: частные лица и организации, заинтересованные в приобретении компьютерной техники — от офисных ПК до игровых станций и рабочей периферии. Также — партнёры и дистрибьюторы в регионе Центральной Азии.

Основные задачи сайта:

  • Представить каталог продукции с фильтрацией и сортировкой;
  • Предоставить конфигуратор для сборки ПК под индивидуальные задачи;
  • Обеспечить удобную корзину для формирования заказа;
  • Реализовать мультиязычность (KZ, RU, EN) и мультивалютность (KZT, USD, RUB);
  • Адаптировать интерфейс для мобильных устройств;
  • Отразить национальный колорит через казахские орнаменты в дизайне.

2.3. Структура ТЗ: разделы, дизайн-требования, функционал

ТЗ включало разработку следующих страниц:

  • Главная страница с каруселью, счётчиками, преимуществами и популярными товарами;
  • «О компании» — история, миссия, ценности, производство;
  • «Каталог» — компьютеры и периферия с фильтрацией и сортировкой;
  • «Компьютеры» — 6 моделей с ценами, характеристиками и возможностью заказа;
  • «Периферия» — 8 товаров (клавиатуры, мыши, наушники, веб-камера, коврик);
  • «Конфигуратор ПК» — пошаговый выбор компонентов с расчётом цены;
  • «Контакты» — адрес, телефон, форма обратной связи, карта;
  • «FAQ» — часто задаваемые вопросы с аккордеоном;
  • «Избранное» — сохранённые пользователем товары;
  • Страница 404 — кастомная страница ошибки.

Дизайн-требования: тёмная тема с синим акцентом, переключение на светлую тему, адаптивность, казахские орнаменты в качестве декоративных элементов.

3. Проектирование пользовательского интерфейса

3.1. Выбор инструментов

Основным инструментом для проектирования интерфейса была выбрана Figma — благодаря возможностям облачного хранения, автосохранения, работы в браузере и удобного экспорта элементов. Figma позволила быстро создавать каркасные схемы (wireframes), продумывать расположение блоков и согласовывать визуальные решения.

3.2. Создание структуры и прототипа

На основе ТЗ была разработана каркасная структура сайта. Определены ключевые блоки для каждой страницы: шапка с навигацией и переключателями языка/валюты, основной контент, подвал с контактами и ссылками. Особое внимание уделено:

  • Логике навигации и пользовательским сценариям (поиск → фильтр → карточка → корзина);
  • Расположению CTA-кнопок и визуальной иерархии;
  • Адаптивности блоков при изменении ширины экрана.

3.3. Подбор цветовой схемы и шрифтов

Основные цвета сайта: тёмный фон (#0f172a), акцентный синий (#3b82f6), текст — белый и серый для вторичной информации. Для светлой темы — обратная схема со светлым фоном и тёмным текстом. Такой контраст обеспечивает комфортное чтение и современный вид. В качестве шрифта выбран Roboto от Google Fonts — читаемый, нейтральный, хорошо подходит для технической тематики.

5. Интеграция библиотек

5.1. Bootstrap 5

Bootstrap 5 был подключён через CDN для использования готовой системы сеток, компонентов (кнопки, карточки, навигация) и утилитарных классов. Это ускорило разработку и обеспечило консистентность элементов. Стили Bootstrap загружаются до основных стилей проекта, что позволяет переопределять их при необходимости. В частности, собственная реализация тёмной темы и казахских орнаментов выполнена поверх Bootstrap.

5.2. Font Awesome

Библиотека иконок Font Awesome 6 подключена для визуального сопровождения элементов интерфейса: иконки поиска, корзины, темы, социальных сетей (Instagram, Facebook, WhatsApp, X/Twitter), контактной информации и преимуществ компании. Иконки загружаются через CDN и кэшируются браузером.

5.3. AOS.js (Animate On Scroll)

Для анимации появления элементов при прокрутке использована библиотека AOS.js. Каждому элементу с классом fade-in программно присваивается атрибут data-aos="fade-up" с stagger-задержкой до 400ms. Анимация происходит однократно (once: true) с длительностью 600ms. Начальное состояние скрыто через CSS (opacity: 0, scale: 0.97), при появлении — плавный переход с cubic-bezier.

9. Трудности, возникшие в процессе работы

9.1. Проблемы в вёрстке и их решения

Одной из первых проблем стала некорректная работа переключателя темы — тема переключалась только один раз из-за того, что переменная currentTheme не обновлялась внутри функции applyTheme. Решение: явно присваивать currentTheme при каждом вызове. Вторая проблема — конфликт класса bg-secondary с Bootstrap, который использует !important для своих фоновых цветов. Решение: добавить !important к нашему CSS-правилу.

9.2. Конфликты библиотек

При подключении Bootstrap 5 его классы переопределили некоторые собственные стили (например, bg-secondary, кнопки). Решение: переопределение через более специфичные селекторы (.section.bg-secondary) и использование !important где необходимо. Также пришлось адаптировать JS-код переключения темы для работы с Font Awesome иконками вместо текстовых эмодзи.

9.3. Сложности с адаптивностью и JS

На мобильных устройствах меню перекрывало контент — решено через position: fixed для меню и z-index. Конфигуратор ПК требовал динамического расчёта цены при изменении любого компонента — реализовано через обработчики событий onChange на всех селектах. Проблема с казахскими орнаментами: изначально узоры выглядели слишком ярко — решено через opacity: 0.04 для фонового паттерна.
